# v0.3.21

## Summary

This release fixes a routing correctness bug in `domain_mapper` when one domain matches multiple overlapping rule types at the same time.

- `full:` and `domain:` matches now continue merging matching `keyword:` and `regexp:` rules
- overlapping policy sets keep their intended combined effect instead of being cut short by early matcher return behavior
- existing YAML configs continue to work without changes

## Fixed

- fixed `domain_mapper` so a `full:` or `domain:` hit no longer prevents matching `keyword:` and `regexp:` rules from being merged into the final result
- fixed real-world overlap cases where one domain belongs to multiple policy groups at once, including white/black/grey lists, redirect domains, ddns domains, dedicated upstream domains, and client/domain driven policy combinations
- restored expected behavior for maintained configurations that rely on configuration ordering and combined rule effects to choose final direct or proxy routing

## Config impact

This release does **not** require YAML changes.

Existing deployments can update only the binary. If a domain was previously written into generated direct/proxy cache files under the old logic, clear the stale generated lists once after upgrade so the fixed matcher can rebuild them correctly.
