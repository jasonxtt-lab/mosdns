# v0.3.23

## Summary

This release fixes audit effective-tag correction for `_exit` sequence variants used by runtime flow.

- `sequence_fakeip_addlist_exit` now receives the same direct-candidate correction behavior as `sequence_fakeip_addlist`
- query logs and overview labels are now consistent when final upstream is proxy but domain-set contains direct-candidate tags

## Fixed

- extended `computeEffectiveTag` sequence check to include `_exit` variant names
- avoided label mismatch cases such as:
  - expected: `记忆直连转代理`
  - previously shown in some `_exit` paths: `记忆直连`

## Tests

- added `_exit` variant regression tests in `coremain/audit_test.go`

## Config impact

This release does **not** require YAML changes.

Existing deployments can update only the binary.
