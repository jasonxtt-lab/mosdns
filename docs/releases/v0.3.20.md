# v0.3.20

## Summary

This release polishes the maintained Vue UI and ships a small build-pipeline improvement for embedded frontend assets.

- overview KPI spacing now adapts better as total query counts grow
- secondary-menu and ordinary-button hover states use color-only feedback without vertical movement
- clickable UI controls keep the normal cursor style for a consistent desktop experience
- the local-rules editor keeps its rounded corner clean when the scrollbar is at the top

## Changed

- refined overview header layout so `总查询数` and `平均处理时间` remain visually separated with wider values
- standardized non-primary button hover behavior under normal and custom-background modes
- removed the hand-cursor override attempt and kept pointer styling consistent as the normal cursor
- added automatic frontend asset query stamping in `webui-log/vite.config.js` for both embedded `index.html` and root `/` `log.html`

## Fixed

- fixed the local-rules editor scrollbar thumb/track edge overlapping the top-right rounded corner
- tightened the local-rules footer hint/status presentation so the hint sits in a compact glass-style chip

## Config impact

This release does **not** require YAML changes.

Existing deployments can update only the binary. If a browser still shows old UI behavior after upgrade, refresh the page once to load the new embedded frontend asset query version.
