# v0.3.22

## Summary

This release focuses on maintained Vue UI workflow consistency and operator safety for upstream proxy behavior.

- all maintained Vue pages now use the top-right compact notice area instead of inline full-width message bars
- local-rules editing now supports cross-list drafts with one-click save-all
- `foreign` upstream entries now fall back to global SOCKS5 when per-item socks5 is empty

## Changed

- unified success/error notice behavior to `mosdns-top-notice` across maintained Vue modules
- adjusted top notice width/placement to avoid covering the header title and to keep message width compact
- local-rules page now keeps unsaved edits while switching list tabs inside the same page session
- local-rules save action now persists all changed lists in one run

## Backend

- `GetUpstreamOverrides("foreign")` now injects global `/api/v1/overrides` socks5 into empty per-item socks5 fields at runtime
- explicit per-item socks5 remains higher priority than global fallback
- added regression tests in `coremain/api_upstream_test.go`

## Config impact

This release does **not** require YAML changes.

Existing deployments can update only the binary. If UI still shows old behavior after upgrade, refresh browser cache once.
