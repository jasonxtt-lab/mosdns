# v0.3.19

## Summary

This release cleans up runtime JSON state placement and fixes the maintained Vue UI overview table behavior.

- runtime state JSON files are now stored under `/cus/mosdns/state`
- the binary automatically migrates existing root-level state files on startup
- the overview `最慢查询` table keeps long domains compact while preserving the `ms` unit on desktop and mobile layouts

## Changed

- moved these generated state files out of `/cus/mosdns` root:
  - `appearance_settings.json`
  - `appearance_text_settings.json`
  - `appearance_button_settings.json`
  - `audit_settings.json`
  - `webui_port_settings.json`
- `config_overrides.json`, `upstream_overrides.json`, and `special_upstream_groups.json` remain in the runtime root and are not migrated
- `requery` now reads the WebUI port override through the same core backend path as the main server

## Fixed

- fixed a desktop overview regression where the `最慢查询` duration cell could show only the number and clip the `ms` unit
- preserved the previous mobile compatibility fixes for long domains and narrow-card layouts

## Config impact

This release does **not** require YAML changes.

Existing deployments can update only the binary. On first start, the binary creates `/cus/mosdns/state` and moves the supported generated state JSON files there automatically.
